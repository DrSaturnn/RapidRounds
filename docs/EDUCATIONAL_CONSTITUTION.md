# RapidRounds Educational Constitution v1.0

RapidRounds teaches clinical reasoning for NBME shelf exams and USMLE Step 2 CK.

It does not merely quiz facts.

The educational goal is transfer:

The learner should recognize the same illness script across unfamiliar NBME-style presentations.

## Core Educational Unit

The core unit is not a question.

The core unit is an illness script.

Each illness script can generate multiple reasoning variants:

- recognition
- diagnosis
- next best step
- stable management
- unstable management
- lab interpretation
- imaging interpretation
- contraindications
- complications
- follow-up
- decision boundary

## Expert Reasoning Sequence

Every case should support:

1. Identify the clinical pattern
2. Find the pivot clue
3. Ignore the noise
4. Commit to the most testable diagnosis or next step
5. Compare against the nearest distractor
6. Store a durable rule

## NBME Alignment

RapidRounds should emphasize:

- illness scripts
- pivot clues
- discriminators
- next-best-step reasoning
- management thresholds
- common distractors
- contraindications
- complications
- testable edge cases

RapidRounds should not emphasize:

- trivia
- excessive pathophysiology before the answer
- passive reading
- long explanations without retrieval
- generic tutoring

## Partial Credit

A learner can be partially correct.

Examples:

- correct broad syndrome, wrong diagnosis
- correct diagnosis, wrong next step
- correct management class, wrong timing
- correct differential, missed pivot clue

The app should identify what was right before repairing what was wrong.

## Decision Repair

Wrong answers should be classified by reasoning failure when possible.

Common failure types:

- diagnosis recognition failure
- decision boundary failure
- management sequencing failure
- lab interpretation failure
- imaging interpretation failure
- contraindication failure
- complication recognition failure
- distractor overweighting
- premature closure
- knowledge gap

The learner should see plain-language explanations, not raw labels.

## Decision Boundary Repair

When two diagnoses or management options are commonly confused, RapidRounds should show the discriminator.

Example:

Placenta previa vs placental abruption

The point is not to show a table.

The point is to teach:

"This is the exact clue that should have changed your answer."

## Teach Me More

Teach Me More exists for optional depth.

It should not interrupt the core reasoning loop.

It should reinforce the current illness script and clarify why the NBME tests this distinction.

## Adaptive Learning

The engine should choose the next case based on:

- current answer
- partial-credit result
- failure type
- decision boundary
- prior learner state
- script mastery
- variant mastery

The learner should feel the app knows exactly what they need next.

## External Question Practice

RapidRounds does not replace UWorld, AMBOSS, CMS, or NBME forms.

Long term, it should convert external misses into targeted reasoning repair.

Do not store copyrighted question stems.

Store learner summaries and reasoning patterns instead.

## Educational Definition of Done

A completed case should leave the learner with:

- one recognized illness script
- one pivot clue
- one repaired confusion
- one durable board-relevant rule
- one next opportunity to transfer the reasoning
